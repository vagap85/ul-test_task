## Чек-лист контроллера orders

### GET /orders (список)

| Сценарий | Ожидаемый код | Пояснение |
|----------|---------------|-----------|
| Успешный запрос, есть записи | 200 OK | Возвращает массив заказов |
| Успешный запрос, нет записей | 200 OK | Возвращает пустой массив [] |
| Нет токена авторизации | 401 Unauthorized | Требуется аутентификация |
| Токен есть, но недостаточно прав | 403 Forbidden | Доступ запрещён |
| Пагинация (page=2, limit=10) | 200 OK | Заголовки: Link, X-Total-Count |
| БД недоступна | 503 Service Unavailable | Retry-After |

### GET /orders/{id} (получение одной записи)

| Сценарий | Ожидаемый код | Пояснение |
|----------|---------------|-----------|
| Заказ существует, не удалён | 200 OK | Возвращает объект заказа |
| Заказ не существует | 404 Not Found | — |
| Заказ логически удалён (был DELETE) | 410 Gone | Парный метод DELETE существует |
| Заказ существует, но принадлежит другому пользователю | 403 Forbidden | — |

### HEAD /orders/{id} (проверка существования)

| Сценарий | Ожидаемый код | Пояснение |
|----------|---------------|-----------|
| Заказ существует | 204 No Content | Без тела ответа |
| Заказ не существует | 404 Not Found | — |

### POST /orders (создание без указания ID)

| Сценарий | Ожидаемый код | Пояснение |
|----------|---------------|-----------|
| Валидный JSON, авторизован | 201 Created | Location: /orders/{newId} |
| JSON сломан (синтаксис) | 400 Bad Request | { "error": "Invalid JSON" } |
| Обязательное поле отсутствует | 422 Unprocessable Entity | { "field": "weight", "error": "required" } |
| Двойной запрос (idempotency key) | 409 Conflict | — |
| Связанный товар не найден | 404 Not Found | — |

### POST /orders/{id:uuid} (создание с клиентским ID)

| Сценарий | Ожидаемый код | Пояснение |
|----------|---------------|-----------|
| ID не существует, данные валидны | 201 Created | — |
| ID уже существует | 409 Conflict | Конфликт, ID занят |

### PUT /orders/{id} (обновление — Update)

| Сценарий | Ожидаемый код | Пояснение |
|----------|---------------|-----------|
| Успешное обновление, возвращаем read-model | 200 OK | Тело с обновлённым объектом |
| Успешное обновление, тело не нужно | 204 No Content | — |
| ETag/If-Match не совпадает | 412 Precondition Failed | Кто-то уже изменил |
| Заказ не существует | 404 Not Found | — |
| Заказ логически удалён | 410 Gone | — |

### PUT /orders/{id} (создание или обновление — Upsert)

| Сценарий | Ожидаемый код | Пояснение |
|----------|---------------|-----------|
| Записи не было → создана | 201 Created | — |
| Запись была → обновлена (с возвратом тела) | 200 OK | — |
| Запись была → обновлена (без тела) | 204 No Content | — |

### DELETE /orders/{id}

| Сценарий | Ожидаемый код | Пояснение |
|----------|---------------|-----------|
| Успешное логическое удаление | 204 No Content | — |
| Заказ не существует | 404 Not Found | — |
| Заказ уже был удалён | 410 Gone | Повторный DELETE |
| Нет прав на удаление | 403 Forbidden | — |

## Общие проверки (сквозные)

| Проверка | Ожидаемый код | Пояснение |
|----------|---------------|-----------|
| Превышен rate limit | 429 Too Many Requests | Retry-After |
| Неверный HTTP метод (PATCH вместо PUT) | 405 Method Not Allowed | Allow: GET, POST, PUT, DELETE, HEAD |
| Content-Type не JSON | 415 Unsupported Media Type | — |
| БД недоступна | 503 Service Unavailable | Не 500 |
