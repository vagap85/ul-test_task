## 1. Что я проверил бы в первую очередь (чек-лист)

### Happy Path (всё хорошо)

- [ ] Создать черновик → заполнить всё → сохранить (ок)
- [ ] Подписать КЭП грузоотправителя → статус сменился на "Подписан ГО"
- [ ] Перевозчик подписывает → "В пути"
- [ ] Грузополучатель подписывает → "Доставлен"
- [ ] Проверить, что ФНС-валидация проходит (в тестовом контуре)

## 2. XSD-проверки (ключевые поля)

| Поле | Обязательность | Границы |
|------|---------------|---------|
| ИНН | Да | 10 или 12 цифр |
| ДатаПогрузки | Да | >= сегодня |
| ВесБрутто | Да | 0.1 – 40000 кг |
| Объём | Нет | 0.01 – 82 м³ |
| Госномер ТС | Условно | regex: А123ВЕ777 |
| ТТН_ссылка | Нет | если есть — валидный URL |

## 3. Негативка

| Действие | Ожидание |
|----------|----------|
| Вес = 0 кг | Ошибка "Вес должен быть > 0" |
| Вес = 45000 кг | Предупреждение (нужно разрешение) |
| Дата погрузки = вчера | Ошибка |
| Подписать перевозчиком раньше ГО | Ошибка |
| Два менеджера открыли черновик | Блокировка |

## 4. Баги

### Баг №1 (Critical)

**Заголовок:** ЭЦП грузополучателя не проверяется на уполномоченность при самоприемке

**Шаги:**
1. Создать ЭТрН, где ГО = ГП = ООО "Ромашка"
2. ГО подписывает своей КЭП
3. Перевозчик → "В пути"
4. ГП подписывает "Доставлен" другой ЭЦП (главбуха)

**Факт:** Система принимает

**Ожидание:** Требовать ту же ЭЦП или проверять полномочия

### Баг №2 (Minor)

**Заголовок:** Объём груза можно ввести отрицательный

**Шаги:**
1. Открыть создание ЭТрН
2. В поле "Объём" ввести -50
3. Отправить

**Факт:** Уходит в ФНС с -50

**Ожидание:** Валидация > 0, сообщение на форме
